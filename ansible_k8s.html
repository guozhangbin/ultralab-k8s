Kubespray 部署 Kubernetes 集群指南

1. Kubespray 简介

Kubespray 是一个基于 Ansible 的 Kubernetes 集群部署工具，它可以帮助你快速、可靠地部署生产级别的 Kubernetes 集群。Kubespray 支持多种容器运行时（Docker、containerd、CRI-O）和网络插件（Calico、Flannel、Cilium 等），并且可以根据你的需求进行灵活配置。

2. 环境准备

2.1 硬件要求

  • 控制平面节点：至少 2GB 内存，2 个 CPU 核心
  • 工作节点：至少 1GB 内存，1 个 CPU 核心
  • 存储：每个节点至少 20GB 磁盘空间

2.2 软件要求

  • Ansible 2.15 或更高版本
  • Python 3.9 或更高版本
  • 目标节点上的操作系统：Ubuntu 20.04/22.04，CentOS 7/8，Debian 10/11 等

2.3 网络要求

  • 所有节点之间可以互相通信
  • 控制平面节点的 6443 端口可以被其他节点访问
  • 所有节点可以访问互联网（或内部镜像仓库）

3. 配置国内镜像源

由于网络原因，建议配置国内镜像源以加速部署过程。

3.1 修改下载源配置

编辑 submodules/kubespray/roles/kubespray-defaults/defaults/main/download.yml 文件：

# gcr and kubernetes image repo define
gcr_image_repo: "gcr.m.daocloud.io"
kube_image_repo: "k8s.m.daocloud.io"

# docker image repo define
docker_image_repo: "docker.m.daocloud.io"

# quay image repo define
quay_image_repo: "quay.m.daocloud.io"

# github image repo define (ex multus only use that)
github_image_repo: "ghcr.m.daocloud.io"

# Download URLs
dl_k8s_io_url: "https://kubernetes-release.pek3b.qingstor.com"

3.2 配置 Docker 镜像加速器

编辑 submodules/kubespray/roles/container-engine/docker/defaults/main.yml 文件：

# Docker registry mirrors
docker_registry_mirrors:
  - "https://docker.m.daocloud.io"
  - "https://registry.docker-cn.com"
  - "https://hub-mirror.c.163.com"
  - "https://docker.aityp.com/"

3.3 修改 Kubernetes 组件镜像配置

编辑 submodules/kubespray/inventory/sample/group_vars/k8s_cluster/k8s-cluster.yml 文件：

# 使用国内镜像加速 Kubernetes 组件下载
kube_apiserver_image_repo: "{{ kube_image_repo }}/kube-apiserver"
kube_controller_manager_image_repo: "{{ kube_image_repo }}/kube-controller-manager"
kube_scheduler_image_repo: "{{ kube_image_repo }}/kube-scheduler"
kube_proxy_image_repo: "{{ kube_image_repo }}/kube-proxy"
etcd_image_repo: "{{ kube_image_repo }}/etcd"
coredns_image_repo: "{{ kube_image_repo }}/coredns/coredns"
pause_image_repo: "{{ kube_image_repo }}/pause"

# 确保 pod_infra_image_repo 使用国内镜像
pod_infra_image_repo: "{{ pause_image_repo }}"

# 使用 QingStor（青云）镜像下载 Kubernetes 二进制文件
kubelet_download_url: "https://kubernetes-release.pek3b.qingstor.com/release/{{ kube_version }}/bin/linux/{{ image_arch }}/kubelet"
kubeadm_download_url: "https://kubernetes-release.pek3b.qingstor.com/release/{{ kube_version }}/bin/linux/{{ image_arch }}/kubeadm"
kubectl_download_url: "https://kubernetes-release.pek3b.qingstor.com/release/{{ kube_version }}/bin/linux/{{ image_arch }}/kubectl"

4. 部署 Kubernetes 集群

4.1 准备 inventory 文件

复制示例 inventory 文件并修改：

cp -r submodules/kubespray/inventory/sample inventory/mycluster

编辑 inventory/mycluster/hosts.yaml 文件，配置你的节点信息：

all:
  hosts:
    master1:
      ansible_host: 192.168.0.200
      ip: 192.168.0.200
      access_ip: 192.168.0.200
    node1:
      ansible_host: 192.168.0.201
      ip: 192.168.0.201
      access_ip: 192.168.0.201
  children:
    kube_control_plane:
      hosts:
        master1:
    kube_node:
      hosts:
        master1:
        node1:
    etcd:
      hosts:
        master1:
    k8s_cluster:
      children:
        kube_control_plane:
        kube_node:
    calico_rr:
      hosts:

4.2 运行部署脚本

使用 Ansible 运行部署脚本：

ansible-playbook -i inventory/mycluster/hosts.yaml submodules/kubespray/cluster.yml -b

4.3 监控部署过程

部署过程中，Ansible 会显示每个任务的执行状态。如果遇到错误，它会停止并显示错误信息。你可以根据错误信息进行排查和修复，然后重新运行部署脚本。

5. 常见问题及解决方案

5.1 镜像拉取失败

问题：部署过程中，容器运行时无法拉取镜像，出现网络超时错误。

解决方案： 1. 确保你已经正确配置了国内镜像源（参考第 3 节） 2. 检查节点的网络连接是否正常 3. 尝试手动拉取镜像，验证镜像源是否可用： bash sudo docker pull k8s.m.daocloud.io/pause:3.10

5.2 RBAC 权限问题

问题：部署完成后，kubernetes-admin 用户无法执行操作，出现权限错误。

解决方案： 1. 检查 kubernetes-admin 用户的集群角色绑定： bash sudo kubectl --kubeconfig=/etc/kubernetes/admin.conf get clusterrolebindings | grep admin 2. 如果没有绑定到 cluster-admin 角色，添加绑定： bash sudo kubectl --kubeconfig=/etc/kubernetes/admin.conf create clusterrolebinding kubernetes-admin-cluster-admin --clusterrole=cluster-admin --user=kubernetes-admin

5.3 Pause 镜像冲突

问题：kubelet 尝试拉取 registry.k8s.io/pause:3.10 镜像，而不是使用国内镜像源。

解决方案： 1. 检查 kubeadm-setup.yml 文件中的 pause 镜像拉取和标记任务： yaml - name: Kubeadm | Pull and tag pause image command: >- /usr/local/bin/ctr image pull {{ pod_infra_image_repo }}:{{ pod_infra_image_tag }} && /usr/local/bin/ctr image tag {{ pod_infra_image_repo }}:{{ pod_infra_image_tag }} registry.k8s.io/pause:{{ pod_infra_image_tag }} become: true when: - inventory_hostname == first_kube_control_plane - not kubeadm_already_run.stat.exists 2. 确保 pod_infra_image_repo 变量指向国内镜像源： yaml pod_infra_image_repo: "{{ pause_image_repo }}" pause_image_repo: "{{ kube_image_repo }}/pause"

5.4 Containerd 配置问题

问题：containerd 无法正确加载 sandbox_image 配置。

解决方案： 1. 重新生成 containerd 的配置文件： bash sudo containerd config default > /etc/containerd/config.toml 2. 修改配置文件中的 sandbox 选项： toml [plugins."io.containerd.cri.v1.images"] sandbox = "k8s.m.daocloud.io/pause:3.10" 3. 重启 containerd 服务： bash sudo systemctl restart containerd

5.5 Kubelet 配置问题

问题：kubelet 尝试拉取 registry.k8s.io/pause:3.10 镜像，而不是使用国内镜像源。

解决方案： 1. 修改 kubelet 的环境变量文件： bash sudo vi /etc/kubernetes/kubelet.env 2. 添加 --pod-infra-container-image 参数： bash KUBELET_ARGS="--bootstrap-kubeconfig=/etc/kubernetes/bootstrap-kubelet.conf \ --pod-infra-container-image=k8s.m.daocloud.io/pause:3.10 \ --config=/etc/kubernetes/kubelet-config.yaml \ --kubeconfig=/etc/kubernetes/kubelet.conf \ --runtime-cgroups=/system.slice/containerd.service \ " 3. 重启 kubelet 服务： bash sudo systemctl restart kubelet

6. 验证集群状态

6.1 检查 API 服务器

sudo netstat -tulpn | grep 6443

如果 API 服务器正常运行，你应该看到 6443 端口在监听。

6.2 检查节点状态

sudo kubectl --kubeconfig=/etc/kubernetes/admin.conf get nodes

如果节点状态为 Ready，则表示节点已经成功加入集群。

6.3 检查控制平面组件

sudo kubectl --kubeconfig=/etc/kubernetes/admin.conf get pods -n kube-system

如果所有控制平面组件（kube-apiserver、kube-controller-manager、kube-scheduler、etcd）的状态为 Running，则表示控制平面已经成功启动。

6.4 检查集群版本

sudo kubectl --kubeconfig=/etc/kubernetes/admin.conf version

这将显示客户端和服务器的 Kubernetes 版本。

7. 总结

Kubespray 是一个功能强大的 Kubernetes 集群部署工具，它可以帮助你快速、可靠地部署生产级别的 Kubernetes 集群。通过配置国内镜像源，你可以加速部署过程，避免网络超时错误。如果遇到问题，你可以参考本文档中的常见问题及解决方案进行排查和修复。

部署完成后，你可以使用 kubectl 命令管理你的 Kubernetes 集群，例如创建、删除和管理 Pod、Deployment、Service 等资源。

祝你部署顺利！
